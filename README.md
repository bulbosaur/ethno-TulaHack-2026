# ethno-TulaHack-2026

Этот сайт был разработан с одной важной целью: сделать изучение народных промыслов России доступным, интересным и по-настоящему увлекательным. Мы живём в мире, где информация окружает нас повсюду, но далеко не всегда она вовлекает. «ЭТНО» решает эту проблему через игровой формат.

Посетителям сайта предлагается не просто читать или смотреть — а взаимодействовать. Через игровые механики пользователь погружается в мир традиционной культуры: знакомится с народами России, узнаёт об их особенностях, традициях и, конечно, ремёслах.

## Структура проекта

Проект представляет собой полноценное веб-приложение с разделением на backend и frontend, построенное с использованием чистого Go.

Ядро приложения организовано по принципу чистой архитектуры с четким разделением ответственности:

```bash
.
├── backend/                     # Исходный код серверной части
│   ├── cmd/                     # Точки входа приложения (main.go)
│   ├── db/                      # Инициализация базы данных (PostgreSQL)
│   ├── internal/                # Основная логика
│   │   ├── auth/                # Авторизация (JWT, хеширование паролей)
│   │   ├── config/              # Загрузка конфигурации
│   │   ├── folks/               # Парсер для обработки JSONB данных этносов на интерактивной карте
│   │   ├── models/              # Модели структур данных
│   │   ├── quest/               # Парсер для обработки JSONB данных квестов
│   │   ├── repository/          # Слой доступа к данным, инкапсулирующий все SQL-запросы
│   │   └── transport/http/      # HTTP-обработчики (handlers, middleware)
│   └── migrations/              # Файлы миграций базы данных
│
├── frontend/                    # Клиентская часть
│   ├── static/                  # Статические файлы
│   │   ├── assets/              # Изображения и медиа
│   │   ├── css/                 # Стили
│   │   └── js/                  # Клиентские скрипты
│   └── templates/               # HTML-шаблоны
│
├── docker-compose.yaml          # Настройки Docker Compose
├── seed.sql                     # Данные для заполнения БД
└── .env_example                 # Пример переменных окружения
```

## Запуск из Docker контейнера

1. Из корня репозитория запустите все сервисы в фоновом режиме:

```bash
docker compose up -d --build
```

2.  Ждём 5–15 сек, пока PostgreSQL завершит инициализацию

3.  Создаём дамп для корректного изображения интерактивной карты и квестов

Linux/macOS

```bash
docker compose exec -T db pg_dump -U dev -d ethno --format=plain --encoding=UTF8 --no-owner --no-acl --clean --if-exists > seed.sql
```

Windows

```bash
docker compose exec -T db pg_dump -U dev -d ethno --format=plain --encoding=UTF8 --no-owner --no-acl --clean --if-exists | Set-Content -Encoding UTF8 seed.sql
```
